## Directory Structure

```
mdaf/
├── ai_engine/
│   ├── dataset.py            # Synthetic data generation (rule engine, n=2000) + MSCDataset
│   ├── encoders.py           # M1~M5 modality encoders (Morph/Metabolic/Donor/Flow/Manuf)
│   ├── cross_attention.py    # Bidirectional Cross-Attention × 6 pairs (h=8) + FusionFFN
│   ├── model.py              # MDAFModel: encoders → projection → fusion → MQS + grade
│   ├── train.py              # Training loop (AdamW + CosineAnnealing + Early Stopping)
│   ├── evaluate.py           # Test evaluation + baseline comparison + ablation study
│   ├── shap_analysis.py      # KernelSHAP XAI — modality & feature-level contributions
│   └── digital_twin.py       # Patient-batch MatchScore (4 φ components + safety penalty)
│
├── zkp/
│   ├── zkp_simulator.py      # Groth16 REP/PQP circuit simulation + benchmark
│   └── certificate.py        # OnChainCertificate + OffChainDataPackage builder
│
├── blockchain/
│   ├── blockchain_simulator.py   # Pure-Python QBFT ledger + 3 smart contract simulators
│   ├── contracts/
│   │   └── MDAPContracts.sol     # Solidity: CertificateRegistry, QualityGateway, LifeBankDirectory
│   └── besu_4node/               # ★ Real 4-node Hyperledger Besu QBFT network
│       ├── __init__.py
│       ├── docker-compose.yml    # 4 validators (172.20.0.11~14), ports 8545~8549
│       ├── besu_client.py        # web3.py client: connect, deploy, transact, measure
│       ├── besu_adapter.py       # Auto-switch: real Besu ↔ simulation fallback
│       ├── benchmark_besu.py     # TPS / finality / gas empirical measurement suite
│       ├── config/               # (auto-generated by setup_network.py)
│       │   ├── genesis.json          # QBFT genesis with RLP-encoded extraData
│       │   ├── node_keys.json        # 4 validator key pairs
│       │   ├── static-nodes.json     # enode peer list
│       │   └── config_nodeN.toml     # Per-node Besu config
│       ├── data/
│       │   └── nodeN/key         # Per-node private key files (Besu format)
│       ├── logs/                 # Docker log output directory
│       └── scripts/
│           ├── generate_keys.py      # Deterministic ECDSA key generation (4 validators)
│           ├── generate_genesis.py   # QBFT genesis.json builder (RLP extraData)
│           └── setup_network.py      # One-shot network initializer (run once)
│
├── data/                         # (auto-generated) synthetic_msc.csv, cnn_embeddings.npy
├── checkpoints/                  # (auto-generated) best_model.pt, history.json
├── outputs/                      # Pipeline report JSON, benchmark results
├── tests/
│   └── test_pipeline.py          # End-to-end integration test (9 steps)
└── scripts/
    └── run_full_pipeline.py      # Single entry point: data→train→eval→ZKP→blockchain→twin
```

## Quick Start

### Option A — Simulation only (no Docker required)

```bash
cd mdaf
python scripts/run_full_pipeline.py
# or
python tests/test_pipeline.py
```

### Option B — Real 4-node Besu QBFT (empirical measurement)

```bash

# Docker 환경
PC에서 "Docker Desktop" 실행
docker compose down --volumes (기존컨테이너 삭제)

# Step 1: Initialize network config (run once)

cd mdaf/blockchain/besu_4node
python scripts/setup_network.py

# Step 2: Start 4-node Docker network
cd blockchain\besu_4node
docker compose up -d     

OR
docker compose -f blockchain\besu_4node\docker-compose.yml up -d

확인사항
:: 4개 컨테이너 모두 Up 확인
docker compose ps
:: 블록 생성 확인 (약 20초 후)
docker compose logs validator1 --tail 20
:: JSON-RPC 응답 확인
curl http://localhost:18545 -X POST -H "Content-Type: application/json" ^
  -d "{\"jsonrpc\":\"2.0\",\"method\":\"eth_blockNumber\",\"params\":[],\"id\":1}"


# Step 3: Wait ~20s for nodes to sync, then run pipeline
python tests/test_pipeline.py
cd ../..
python scripts/run_full_pipeline.py --epochs 60 
# → Adapter auto-detects localhost:8545 and switches to real Besu
# --run_qbft_bench : outputs/qbft_benchmark_*.json  ← 논문 Table 15 바로 사용
# Step 4: Run dedicated benchmark
cd blockchain/besu_4node
python benchmark_besu.py --n-certs 20 --n-tx 50
# → outputs/besu_benchmark_results.json
```

### Option C — Benchmark only (after docker compose up)

```bash
cd mdaf/blockchain/besu_4node
python benchmark_besu.py
```

## Blockchain Mode Auto-Switch

The `besu_adapter.py` detects `localhost:8545` at runtime:

```
Docker NOT running                  Docker running
──────────────────────              ──────────────────────────────
[Adapter] ⚠ simulation fallback    [Adapter] ✓ real Besu 4-node
blockchain_simulator.py             web3.py → real QBFT txs
TPS/finality hardcoded              TPS/finality empirically measured
```

**No code change required** — just start/stop Docker.

## Requirements

Python 3.9+

scikit-learn, shap, numpy, pandas, matplotlib, scipy
web3 >= 6.0
eth-account, eth-keys, rlp

            web3 eth-account eth-keys rlp py-solc-x
